JavaScript面向对象编程

发表于 2017-02-18

简述

　　JavaScript是一门基于对象的语言，但是因为语法中没有类(class)，所以并不是一种真正的面向对象编程语言。

关于JavaScript中的this关键字

　　Java中的this指的是当前对象，很容易掌握理解。
　　JavaScript中的this在运行期间才进行绑定，因此其含义丰富得多，可以分为以下几种情况：
　　1. 作为对象方法调用
　　这时函数作为一个对象的属性，this就绑定到当前对象上。

　　2. 作为函数调用
　　如果直接调用函数，此时this绑定到全局对象。

　　假设函数为一个内部函数，但是绑定时依旧会绑定到全局变量，这样就产生一个问题，如下：

　　对象中moveTo方法的目的是想要更改对象中的x和y值，但是由于this对象绑定的是全局对象，导致this.x建立了一个全局变量，而未改变对象中的值。
　　解决方案：

　　使用变量替代的方法，由于var的预编译特性，在执行var that = this语句时，this绑定的是该对象。
　　3. 作为构造函数调用
　　这时通过这个函数，生成一个新对象，this就绑定在了这个新对象上。

　　构造函数约定俗成的规矩是——首字母大写。
　　4. 使用 apply 或 call 调用

　　如果apple或call的参数为空，则默认为全局对象。
