关于localstorage的使用

发表于 2017-03-06

  1. 检查浏览器是否支持

  1. 操作方法

    localStorage 保存的数据，都以“键值对”的形式存在。

  2. 事件监测

    当localstorage中的数据发生变化是，会触发 storage 事件，我们可以指定这个事件的回调函数。

该event对象有4个属性，分别是

  • key：发生变化的键名。
  • oldValue：更新前的值。如果该键为新增加，则这个属性为null。
  • newValue：更新后的值。如果该键被删除，则这个属性为null。
  • url：原始触发storage事件的那个网页的网址。

注意 :除了IE浏览器外，storage事件不会再导致数据变化的页面触发，只会在浏览器打开的同一域名下的其他页面触发，通过这个特性，我们可以利用该事件完成一些简单的多窗口通信。

  1. 缺点与优点

    缺点主要集中在在PC网页上
  • 兼容性不太好，不支持LS的浏览器比例仍然很大。
  • LS读取+eval很多时候速度会比不上304缓存。
  • 浏览器本地缓存足够可靠持久。
    优点都在于移动端网页上
  • 浏览器兼容性友好。
  • 由于移动端通常网速较慢，速度会比304缓存快。
  • 以模块文件为单位，缓存失效率低。
