对于JavaScript作用域的一点理解

发表于 2017-02-17

一句话精辟描述

JavaScript中的函数运行在它们被定义的作用域里,而不是它们被执行的作用域里。

作用域(scope)

  1. 全局作用域(Global Scope)

    　　拥有全局作用域的几种情况

    　　(1) 最外层函数和在最外层函数外面定义的变量
    　　(2) 所有末定义直接赋值的变量
    　　(3) 所有window对象的属性

  2. 局部作用域

    　　例如函数内部。

    作用域链(scope chain)

    　　作用域链采用的是列表的方式，在函数被定义时，会将这时的作用域链链接到函数对象的scope属性中，在发生标识符解析的时候，会逆向查询scope chain列表的每一个活动对象属性，找不到着为undefined。

    代码优化

    　　标识符所在的位置越深，读写速度就会越慢，而全局变量总是存在于作用域链的最末端，因此尽量少使用全局变量。

    如果一个跨作用域的对象被引用了一次以上，则先把它存储到局部变量里再使用。

    　　　这样做可以减少在作用域链中查找变量的时间。

预编译

　　JavaScript虽然是脚本语言，但是是由预编译过程的，以段为处理单元。
　　在执行 每一段 JavaScript代码时，会首先处理 var关键字 与 function定义式 ,即为预编译。
　　注： 函数表达式在执行过程中才计算。

相关知识

  2. 拖拽事件周期中会初始化一个DataTransfer对象,用于保存拖拽数据和交互信息.以下是它的属性和方法.

    dropEffect: 拖拽交互类型,通常决定浏览器如何显示鼠标光标并控制拖放操作.常见的取值有copy,move,link和none
    effectAllowed: 指定允许的交互类型,可以取值:copy,move,link,copyLink,copyMove,limkMove, all, none默认为uninitialized(允许所有操作)
    files: 包含File对象的FileList对象.从操作系统向浏览器拖放文件时有用.
    types: 保存DataTransfer对象中设置的所有数据类型.
    setData(format, data): 以键值对设置数据,format通常为数据格式,如text,text/html
    getData(format): 获取设置的对应格式数据,format与setData()中一致
    clearData(format): 清除指定格式的数据
    setDragImage(imgElement, x, y): 设置自定义图标

以hexo+github搭建自己的个人博客，小白向

发表于 2016-11-28

个人博客适合的人群

  • 喜欢折腾，喜欢拥有自己独立的空间
  • 会使用或者想学习使用markdown语言
  • 愿意在网络上记录自己

前期准备

1. 安装Node.js

安装Node.js是非常简单的事情，在官网选择对应版本下载即可(下载地址)。

2. 安装git

现在安装git也是很简单的事情，同样在官网选择对应版本即可(下载地址)。
git的基本使用也较为简单，在桌面或者自己想要定位的文件夹下点击鼠标右键，如图所示。



git Bash 是命令行界面。

git GUI 则是图形化界面。

在此以用命令行为例。

3. 安装及配置HEXO

桌面右键鼠标，点击Git Bash Here，输入以下命令即可安装

npm install hexo-cli -g

等待安装完成，然后输入：

npm install hexo –save

通过输入一下命令，检查是否安装成功

hexo -v

如果显示了一堆版本信息，则证明安装成功。

接着在你想要安装的文件夹中，点击鼠标右键，点击Git Bash Here，输入以下命令进行初始化

hexo init

npm install

可以看见该文件夹目录如下：

接下来执行

hexo g

hexo s

保持Git Bash窗口不关闭，在localhost:4000网址中就能看见hexo的初始化界面。

4.将Hexo部署到github pages上

  1. 首先，建立一个github pages项目

点击New repository创建新项目。

  1. 配对SSH密钥

配置Github的SSH密钥可以让本地git项目与远程的github建立联系，让我们在本地写了代码之后直接通过git操作就可以实现本地代码库与Github代码库同步。操作如下：

  • 查看是否存在SSH keys，如不存在则需创建

    $ cd ~/. ssh

  • 不存在.ssh目录，则需创建一对新的SSH密钥

    ssh-keygen -t rsa -C “your_email@example.com”

一路按着Enter下去。

  • 将ssh密钥复制到粘贴版钟

    clip < ~/.ssh/id_rsa.pub

  • 登陆github，添加SSH

  • 查看是否成功

    ssh -T git@github.com

如果出现了Hi example你的用户名，则成功了。

  • 最后一步，完善个人信息

    git config –global user.name “cnfeat”//用户名

    git config –global user.email “cnfeat@gmail.com”//填写自己的邮箱

5.将本地Hexo文件更新到github中

  • 复制自己github.io的https地址
  • 粘贴至hexo文件夹下_config.yml文件的”respsitory”中

  • 执行以下指令

    hexo g

    hexo d

  • 完成同步

6.新建MD文章

  • Git bash中输入如下命令建立新文章

    hexo n “你的文章名字”

  • 用markdown编辑器打开该文件进行编辑

  • 可以开着两个Git Bash，让一个一直执行hexo s命令，另一个用于新建文章，这样就能保证文章保存后立马部署到服务器上，接下来只需要同步到github上即可。
